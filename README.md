<h2>🏗️ Monorepo Architecture</h2>

<br/>

![Nx](https://img.shields.io/badge/Nx-143055?style=for-the-badge&logo=nx&logoColor=white)
![Turborepo](https://img.shields.io/badge/Turborepo-EF4444?style=for-the-badge&logo=turborepo&logoColor=white)

<br/>

<table>
<tr>
<td width="50%" valign="top">

#### <img src="https://img.shields.io/badge/Nx-143055?style=flat-square&logo=nx&logoColor=white" /> Nx — Smart Monorepo Orchestrator

Nx is an **intelligent build system** with first-class monorepo support. It analyzes your project graph to run only what's affected, caching everything for speed.

**What I use it for:**
- 🔀 **Project Graph** — auto-detects dependencies between apps & libs
- ⚡ **Computation Caching** — never rebuild unchanged code twice
- 🎯 **Affected Commands** — `nx affected:build` runs only what changed
- 🧩 **Generators** — scaffold components, libs, and modules consistently
- 📦 **Publishable Libraries** — share packages across apps with versioning
- 🏷️ **Tags & Boundaries** — enforce architectural rules (`scope:frontend` can't import `scope:backend`)
- ☁️ **Nx Cloud** — distributed caching & task execution across CI agents

```
my-workspace/
├── apps/
│   ├── web/          # Next.js app
│   ├── admin/        # Nuxt.js dashboard
│   └── api/          # NestJS backend
├── packages/
│   ├── ui/           # Shared component library
│   ├── utils/        # Common utilities
│   └── config/       # Shared ESLint, TS configs
├── nx.json
└── project.json
```

</td>
<td width="50%" valign="top">

#### <img src="https://img.shields.io/badge/Turborepo-EF4444?style=flat-square&logo=turborepo&logoColor=white" /> Turborepo — High-Performance Build System

Turborepo is a **blazing-fast build system** by Vercel that makes monorepo pipelines simple with zero config and maximum speed.

**What I use it for:**
- 🚀 **Pipeline Definition** — declare task dependencies in `turbo.json`
- 💨 **Remote Caching** — share build cache across team & CI (Vercel)
- 🔄 **Parallel Execution** — runs independent tasks concurrently
- 📊 **Task Graph** — `turbo run build --graph` visualizes pipeline
- 🎯 **Scoped Filtering** — `turbo run build --filter=web...` builds only what's needed
- 🔁 **Incremental Builds** — hash-based cache invalidation
- 🪶 **Zero Config** — works with any package manager (npm, pnpm, yarn)

```
my-turborepo/
├── apps/
│   ├── web/          # Next.js frontend
│   ├── docs/         # Documentation site
│   └── api/          # Express/NestJS server
├── packages/
│   ├── ui/           # Shared React components
│   ├── database/     # Prisma schema & client
│   ├── eslint-config/# Shared ESLint config
│   └── typescript-config/ # Shared tsconfig
├── turbo.json        # Pipeline configuration
└── pnpm-workspace.yaml
```

</td>
</tr>
</table>

<details>
<summary><b>⚔️ Nx vs Turborepo — When I Use Which</b></summary>
<br/>

<div align="center">

| Feature | Nx | Turborepo |
|---------|:--:|:---------:|
| **Computation Caching** | ✅ Local + Nx Cloud | ✅ Local + Vercel Remote |
| **Task Orchestration** | ✅ Advanced (project graph) | ✅ Pipeline-based |
| **Code Generators** | ✅ Built-in (`nx generate`) | ❌ Use external tools |
| **Affected Analysis** | ✅ `nx affected` | ✅ `--filter` with `...` |
| **IDE Integration** | ✅ VS Code Extension | ⚠️ Basic |
| **Plugin Ecosystem** | ✅ Rich (React, Nest, Next) | ⚠️ Minimal |
| **Learning Curve** | 📈 Steeper | 📉 Simpler |
| **Best For** | Enterprise, polyglot repos | JS/TS focused, Vercel stack |

</div>

> **My approach:** I reach for **Turborepo** when building Vercel-deployed Next.js + NestJS apps (fast setup, pnpm workspaces, remote cache). I use **Nx** for larger enterprise projects that need code generators, strict module boundaries, and multi-language support (e.g. Rust Solana programs alongside TypeScript frontends).

</details>
